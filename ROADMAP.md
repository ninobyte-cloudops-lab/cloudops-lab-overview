# Roadmap

A high-level view of how the AI-Native CloudOps Lab matures. This is an overview only; no internal dates, account details, implementation details, or launch promises are published.

## Current public status

| Area | Public status |
|---|---|
| Product and curriculum direction | Established in the private core repository |
| Live AWS learner access | Not open publicly |
| Paid cohort readiness | Not announced |
| Safety and cost validation | Required before live AWS execution |
| Public materials | Overview-only |

## Phases

### 1. Product architecture
Define the lab's learning outcomes, sandbox model, and governance approach.

### 2. Student onboarding
Design a clear, low-friction path for learners to enter the governed sandbox safely.

### 3. Sandbox validation
Confirm the controlled environment behaves predictably, keeps practice contained, and gives learners enough realism without uncontrolled access.

### 4. Cost and safety validation
Validate cost controls, permission boundaries, service guardrails, logging, and teardown before any live AWS execution is enabled.

### 5. Pilot cohort readiness
Prepare a small, reviewed pilot once the prior phases are validated and learner onboarding is clear.

---

Progression between phases is gated by readiness, not by calendar. Live AWS execution remains gated until cost, safety, permission-boundary, and teardown validation are complete.
